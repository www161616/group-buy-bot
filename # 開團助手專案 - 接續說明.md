# 開團助手專案 - 接續說明

## 我是誰
我是香奈，經營「包子媽生鮮小舖」（丸十水產股份有限公司），做食品團購生意，有16間分店。

## 我的痛點
我每天要開團賣 25-30 樣商品，目前流程是：
1. 供應商在 LINE「找貨群」丟商品資訊（圖片+文字：商品名、價格、規格、結單日）
2. 員工從 LINE 群挑商品，手動排到 Google 日曆（每個商品一個事件，用時間排序）
3. 員工再到「樂樂團購系統」(leletobuy.com) 逐一上架商品

痛點：步驟3 員工要在 Google 日曆、LINE、樂樂三個系統來回切換，每次上架都要重新找圖文、填價格、填款式。新人根本學不來。

## 目前已完成的事

### 1. 樂樂商品資料已匯出
- 已從樂樂後台匯出 5000 筆商品（2025/01/01 ~ 2026/04/03）
- 包含款式明細（每個商品最多20個款式，有名稱、售價、成本）
- Excel 檔案：`樂樂商品匯出_含款式.xlsx`
- 欄位：商品ID, 商品名稱, 售價(最低), 售價(最高), 圖片連結, 款式數, 訂單數, 未配數, 庫存, 上架日, 更新日, 建立時間, 商品編號(feedid), 款式1_名稱, 款式1_售價, 款式1_成本, ... 款式20_名稱, 款式20_售價, 款式20_成本

### 2. 匯出用的 API 和腳本已確認
樂樂後台的關鍵 API：
- `POST /ajax2/Group_get_feed_list.php` — 商品列表（參數：groupid=g55263, page, limit, fromdate, tilldate, filter_type 等）
- `POST /ajax/Group_get_feed_style_n_price.php` — 款式明細（參數：feedid, groupid）
- 回傳格式：JSON，款式在 `data.gfpid_arr[]`，每個有 style, price, cost 欄位
- 頁面上 `table` 全域變數是 DataTable 物件，`table.data()` 可取得所有商品
- 頁面自帶 `XLSX` 函式庫可直接匯出 Excel
- 匯出腳本在 Chrome Console 執行，每筆商品呼叫款式 API，間隔 80ms，5000 筆約 8 分鐘

### 3. Google 日曆讀取已驗證
- 用 Chrome 開 Google 日曆日檢視 (`calendar.google.com/calendar/u/0/r/day`)
- `document.querySelectorAll('[data-eventchip]')` 可抓到所有事件
- 每個事件的 `innerText` 格式：`"09:00 至 09:30，香菜小籠包，日曆：包子媽生鮮小舖，..."`
- 用 regex `text.match(/(\d{1,2}:\d{2})\s*至\s*(\d{1,2}:\d{2})[，,]\s*(.+?)[，,]\s*日曆/)` 可提取時間和商品名
- 今天 (4/3) 測試成功抓到 27 個事件

### 4. 商品比對邏輯
- 日曆上的商品名（如「香菜小籠包」）跟樂樂標題（如「⬜香菜鮮肉小籠包 香菜控專屬口味💰85⏰4/5結單#151226010#」）做模糊比對
- 目前比對率約 37%（10/27），因為很多日曆上的商品是新品，樂樂還沒有
- 商品標題裡有自訂編號如 `#331030010#`，但日曆上不會放編號
